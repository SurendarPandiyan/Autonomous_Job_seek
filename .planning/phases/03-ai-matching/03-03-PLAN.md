# Plan 03-03: Job Embedding Celery Task

## Objective

Create `src/jobplatform/matching/tasks.py` with `embed_jobs_batch`, the Celery task that
backfills embeddings for all jobs that have `embedding IS NULL`. It processes records in
batches of 20 (one OpenAI API call per batch), then writes the 1536-dim vectors back to
`jobs.embedding`. Register the task queue in `celery_app.py` and expose it via
`POST /api/v1/workers/embed-jobs` in the workers router.

Follow the exact `asyncio.run(_async_impl)` pattern from `jobs/tasks.py`.

## Must Haves

- [ ] `from jobplatform.matching.tasks import embed_jobs_batch` succeeds without ImportError
- [ ] `embed_jobs_batch` is a synchronous Celery task with name `"jobplatform.matching.tasks.embed_jobs_batch"`
- [ ] Inner async function opens `AsyncSessionLocal()` as context manager (never passes session across boundary)
- [ ] Processes jobs in batches of 20; sleeps 0.5 s between batches to respect rate limits
- [ ] Returns `{"embedded": N, "skipped": 0}` dict where N is the count of newly embedded jobs
- [ ] `POST /api/v1/workers/embed-jobs` returns 202 with `{"task_id": "..."}` when called
- [ ] `uv run pytest tests/ -q --tb=short` passes

## Tasks

### Task 1: Create matching/tasks.py with embed_jobs_batch

**Files:**
- `src/jobplatform/matching/tasks.py` — new file, created from scratch

**Instructions:**

Create `src/jobplatform/matching/tasks.py` with these exact imports at the top:

```python
import asyncio
import time

import structlog
from sqlalchemy import select

from jobplatform.celery_app import celery_app
from jobplatform.database import AsyncSessionLocal
from jobplatform.jobs.models import Job
from jobplatform.matching.service import build_job_text, get_embedding
```

Define the async implementation function `_embed_jobs_batch_async`:

```python
async def _embed_jobs_batch_async(task_id: str) -> dict:
```

Body:
1. `logger = structlog.get_logger()` (define at module level or inside function — module level is preferred)
2. `embedded = 0`
3. Open `async with AsyncSessionLocal() as db:` for the entire batch loop.
4. Inside, run a loop that:
   a. Fetches up to 20 jobs: `stmt = select(Job).where(Job.embedding.is_(None)).where(Job.is_active == True).limit(20)` then `result = await db.scalars(stmt)` then `batch = list(result.all())`
   b. If `not batch`: break out of the loop.
   c. For each job in batch: call `text = build_job_text(job)`, then `embedding = await get_embedding(text)`, then set `job.embedding = embedding`.
   d. After processing all 20: `await db.commit()` and increment `embedded += len(batch)`.
   e. `logger.info("embed_jobs_batch.progress", embedded=embedded, task_id=task_id)`
   f. `time.sleep(0.5)` — rate limit buffer between batches.
5. Return `{"embedded": embedded, "skipped": 0}`.

Define the Celery task (sync wrapper):

```python
@celery_app.task(name="jobplatform.matching.tasks.embed_jobs_batch", bind=True)
def embed_jobs_batch(self) -> dict:
    task_id = self.request.id or "local"
    return asyncio.run(_embed_jobs_batch_async(task_id))
```

This is the complete file content for now. `embed_profile` and `compute_matches_for_user` will be appended by Plans 03-04 and 03-05.

Define `logger = structlog.get_logger()` at module level, before any function definitions.

**Verify:** `uv run python -c "from jobplatform.matching.tasks import embed_jobs_batch; print(embed_jobs_batch.name)"` prints `jobplatform.matching.tasks.embed_jobs_batch`.
