# Plan 03-06: Match Router and Schemas

## Objective

Create `matching/schemas.py` with the Pydantic response models and `matching/router.py` with
two authenticated GET endpoints. `GET /api/v1/matches/` returns ranked `JobMatch` records for
the authenticated user using cursor pagination (same pattern as jobs/router.py). `GET
/api/v1/jobs/{id}/match` returns the stored match score if it exists, computes it on-demand
if both embeddings are present, or returns 202 with `status=pending` if embeddings are missing.
Register the router in `main.py`.

## Must Haves

- [ ] `GET /api/v1/matches/` returns `list[MatchResponse]` ordered by `score DESC`
- [ ] `GET /api/v1/matches/` supports `?limit=` and `?cursor=` query params
- [ ] `GET /api/v1/jobs/{id}/match` returns 200 with `MatchResponse` when a `JobMatch` row exists
- [ ] `GET /api/v1/jobs/{id}/match` returns 202 with `{"status": "pending", "score": null}` when embeddings are missing
- [ ] `GET /api/v1/jobs/{id}/match` computes score on-demand when embeddings are present but no match record exists yet
- [ ] Both endpoints require valid JWT — unauthenticated requests return 401
- [ ] `from jobplatform.matching.router import router` succeeds without ImportError
- [ ] `uv run pytest tests/ -q --tb=short` passes

**Endpoint 2 — GET /api/v1/jobs/{job_id}/match:**

This endpoint has three branches:

```python
@router.get("/api/v1/jobs/{job_id}/match")
async def get_job_match_endpoint(
    job_id: int,
    current_user: User = Depends(get_current_user),
    db: AsyncSession = Depends(get_db),
):
    # Branch 1: pre-computed match exists → return it
    match = await get_job_match(db, current_user.id, job_id)
    if match:
        return MatchResponse.model_validate(match)

    # Load job and profile to determine if on-demand compute is possible
    job = await db.scalar(select(Job).where(Job.id == job_id))
    if not job:
        raise HTTPException(status_code=404, detail="Job not found")

    profile = await db.scalar(select(Profile).where(Profile.user_id == current_user.id))

    # Branch 2: embeddings missing → return 202 pending
    if job.embedding is None or profile is None or profile.embedding is None:
        return JSONResponse(
            status_code=202,
            content={"status": "pending", "score": None, "job_id": job_id},
        )

    # Branch 3: both embeddings present → compute on-demand
    prefs = await db.scalar(
        select(JobPreferences).where(JobPreferences.user_id == current_user.id)
    )
    profile_text = build_profile_text(profile, prefs)
    profile_embedding = await get_embedding(profile_text)

    similar = await find_similar_jobs(db, profile_embedding, limit=1)
    # find the score for this specific job
    score = next((s for j, s in similar if j.id == job_id), 0.0)

    profile_skills: list[str] = profile.skills or []
    from jobplatform.matching.service import build_job_text
    job_text = build_job_text(job)
    ats = compute_ats_score(profile_skills, job_text)
    gaps = compute_skill_gaps(profile_skills, job.requirements)

    await upsert_match(db, current_user.id, job_id, score, ats, gaps)
    await db.commit()

    match = await get_job_match(db, current_user.id, job_id)
    return MatchResponse.model_validate(match)
```

Note on Branch 3: `find_similar_jobs` with `limit=1` will not return the specific job_id
unless it's the top result. A more correct approach for single-job on-demand scoring is to
compute cosine distance directly. Replace the `similar` lookup with:

```python
    from pgvector.sqlalchemy import Vector
    import numpy as np
    # direct cosine similarity between two vectors (no DB needed)
    import math
    pv = profile_embedding
    jv = job.embedding
    dot = sum(a * b for a, b in zip(pv, jv))
    norm_p = math.sqrt(sum(a * a for a in pv))
    norm_j = math.sqrt(b * b for b in jv)
    score = dot / (norm_p * norm_j) if norm_p and norm_j else 0.0
```

Actually, use the simpler pgvector approach via a DB query scoped to just one job:

```python
    from sqlalchemy import select
    stmt = (
        select((1 - Job.embedding.cosine_distance(profile_embedding)).label("score"))
        .where(Job.id == job_id)
    )
    result = await db.execute(stmt)
    row = result.first()
    score = float(row.score) if row else 0.0
```

Use this DB approach (do NOT use the numpy/zip loop) — it is consistent with the HNSW index
and avoids floating-point Python loops.
