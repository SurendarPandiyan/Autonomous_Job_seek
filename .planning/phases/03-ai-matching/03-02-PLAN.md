# Plan 03-02: JobMatch Model and Alembic Migration

## Objective

Define the `JobMatch` ORM model with `MatchStatus` enum in `matching/models.py`. Patch
`alembic/env.py` so Alembic recognises the `vector` type during autogenerate (prevents it from
dropping existing `embedding` columns). Write `alembic/versions/0006_create_job_matches.py`
by hand (autogenerate cannot produce HNSW indexes) and run `alembic upgrade head` to apply it.

## Must Haves

- [ ] `job_matches` table exists in the database after migration with columns: id, user_id, job_id, score, ats_score, skill_gaps, status, created_at, updated_at
- [ ] Unique constraint `uq_job_matches_user_job` on (user_id, job_id)
- [ ] HNSW index `jobs_embedding_hnsw` on `jobs.embedding` using `vector_cosine_ops`
- [ ] `alembic upgrade head` exits 0
- [ ] `from jobplatform.matching.models import JobMatch, MatchStatus` succeeds without ImportError

### Task 2: Patch alembic/env.py and write + run migration 0006

**Files:**
- `alembic/env.py` — add matching model import + pgvector ischema_names patch
- `alembic/versions/0006_create_job_matches.py` — hand-written migration

**Instructions for alembic/env.py:**

Open the existing `alembic/env.py`. Two changes are required:

1. After the last `import jobplatform.jobs.models` noqa line, add:
   `import jobplatform.matching.models  # noqa: F401 — registers JobMatch with Base.metadata`

2. Replace the existing `run_migrations_online` function with the version below. The critical
   change is that the lambda passed to `run_sync` now patches `conn.dialect.ischema_names`
   before calling `context.configure`, so the dialect recognises the `vector` type and does
   not emit `NullType` for existing `embedding` columns:

```python
async def run_migrations_online() -> None:
    connectable = create_async_engine(_db_url)
    async with connectable.connect() as connection:
        def configure(conn):
            from pgvector.sqlalchemy import Vector
            conn.dialect.ischema_names["vector"] = Vector
            context.configure(conn, target_metadata=target_metadata)
        await connection.run_sync(configure)
        async with connection.begin():
            await connection.run_sync(lambda _: context.run_migrations())
    await connectable.dispose()
```

Do not change any other part of `env.py`.

**Instructions for migration 0006:**

Create `alembic/versions/0006_create_job_matches.py`. Set:
- `revision = "0006"`
- `down_revision = "0005"` (matches the filename of the previous migration `0005_create_jobs.py`)
- `branch_labels = None`
- `depends_on = None`

Imports needed:
```python
import sqlalchemy as sa
from alembic import op
from sqlalchemy.dialects import postgresql
```

`upgrade()` function body — execute in this exact order:
1. `op.create_table("job_matches", ...)` with all columns. Pass columns as positional args to `op.create_table`:
   - `sa.Column("id", sa.Integer(), primary_key=True)`
   - `sa.Column("user_id", sa.Integer(), sa.ForeignKey("users.id", ondelete="CASCADE"), nullable=False)`
   - `sa.Column("job_id", sa.Integer(), sa.ForeignKey("jobs.id", ondelete="CASCADE"), nullable=False)`
   - `sa.Column("score", sa.Float(), nullable=True)`
   - `sa.Column("ats_score", sa.Float(), nullable=True)`
   - `sa.Column("skill_gaps", postgresql.JSONB(astext_type=sa.Text()), nullable=True)`
   - `sa.Column("status", sa.Enum("new", "saved", "applied", "dismissed", name="matchstatus"), nullable=False, server_default="new")`
   - `sa.Column("created_at", sa.DateTime(timezone=True), nullable=False)`
   - `sa.Column("updated_at", sa.DateTime(timezone=True), nullable=False)`
   - `sa.UniqueConstraint("user_id", "job_id", name="uq_job_matches_user_job")`

2. `op.create_index("ix_job_matches_user_id", "job_matches", ["user_id"])`
3. `op.create_index("ix_job_matches_job_id", "job_matches", ["job_id"])`
4. `op.create_index("ix_job_matches_score", "job_matches", ["score"])`
5. HNSW index (hand-written, must use `IF NOT EXISTS`):
   `op.execute("CREATE INDEX IF NOT EXISTS jobs_embedding_hnsw ON jobs USING hnsw (embedding vector_cosine_ops) WITH (m=16, ef_construction=64)")`

`downgrade()` function body:
1. `op.execute("DROP INDEX IF EXISTS jobs_embedding_hnsw")`
2. `op.drop_index("ix_job_matches_score", table_name="job_matches")`
3. `op.drop_index("ix_job_matches_job_id", table_name="job_matches")`
4. `op.drop_index("ix_job_matches_user_id", table_name="job_matches")`
5. `op.drop_table("job_matches")`
6. `op.execute("DROP TYPE IF EXISTS matchstatus")`

After writing both files, run the migration:
`uv run alembic upgrade head`

If the migration fails with "index already exists" for `jobs_embedding_hnsw`, that is OK because we used `IF NOT EXISTS`. Any other error must be diagnosed and fixed.
