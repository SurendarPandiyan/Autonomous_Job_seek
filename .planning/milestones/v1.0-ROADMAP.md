# Milestone v1.0: Job Platform MVP

**Status:** ✅ SHIPPED 2026-06-27

## Overview

Full-stack job aggregation and AI matching platform. Scrapes listings from 4 portals, scores them via OpenAI embeddings + pgvector cosine similarity, and lets users apply with LLM-tailored resumes in one click. Built in ~10 hours across 66 commits, 3,989 LOC.

## Milestone Summary

**Key Decisions:**
- SQLAlchemy async + asyncpg (not sync ORM) — enables non-blocking DB in FastAPI
- Alembic explicit migrations (no autogenerate) — avoids pgvector ischema_names issues
- text-embedding-3-small (1536 dims) — matches Vector(1536) slot prepped in Phase 2
- asyncio.run() wrapper in Celery tasks — bridges sync Celery with async service layer
- claude-haiku-4-5 for resume tailoring — cost-effective, sufficient quality
- 202 Accepted on apply endpoint — never blocks on LLM in request path

**Issues Resolved:**
- pgvector ischema_names patch required in alembic/env.py for migration autogenerate
- Celery session boundary: never pass AsyncSession across sync/async boundary; open new session inside async impl
- DB test isolation: connection-level transaction rollback, patch AsyncSessionLocal per task test

**Issues Deferred:**
- REQ-09 (actual portal submission) — one-click apply records + queues but doesn't POST to external portals; deferred to v1.1
- Portal adapter real scraping — adapters return mock data; real httpx+BS4 scraping deferred
- Resume parsing — user provides structured data directly

**Technical Debt:**
- Portal adapters return mock data (no real scraping yet)
- No rate limiting on API endpoints
- No background job monitoring/observability dashboard
